# Execution Plans

Use checked-in plans for multi-step or high-risk work.

## Active Plans

Create plans in [active](active) with this name pattern:

```text
YYYY-MM-DD-short-slug.md
```

## Completed Plans

Move finished plans to [completed](completed) after validation.

## Required Sections

- Goal.
- Acceptance criteria.
- Context links.
- Steps.
- Validation.
- Decision log.
- Progress log.

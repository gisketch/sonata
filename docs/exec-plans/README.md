# Execution Plans

Use execution plans for work that needs more than a tiny inline plan.

## Active Plans

Create active plans in [active](active) with this naming pattern:

```text
YYYY-MM-DD-short-slug.md
```

## Completed Plans

Move finished plans to [completed](completed) after implementation and verification.

## Required Sections

- Goal.
- Acceptance criteria.
- Context links.
- Steps.
- Validation.
- Decision log.
- Progress log.

## Rule

Plans are working documents. Update them as facts change instead of preserving an outdated original plan.
